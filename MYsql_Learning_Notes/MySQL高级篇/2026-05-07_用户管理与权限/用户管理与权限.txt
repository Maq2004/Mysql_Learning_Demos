1. **登录命令的细节**：在登录 MySQL 时，参数 `-e` 的作用是什么？如果你想在不进入交互界面的情况下查询用户信息，该怎么写命令？

   ​		-e 的作用是在不进入交互界面的情况下向数据库中注入命令

   ​		

   ```mysql
   mysql -uroot -p -e"show databases"
   ```

   ​		   即可显示数据库系统中的数据库

   

   

2. **创建用户的默认行为**：使用 `CREATE USER 'zhang3' IDENTIFIED BY '123123';` 创建用户时，如果不指定主机名，默认的 Host 是什么？这意味着什么？   

   ​		默认是'%' , 意味着不限制ip登录。

   

3. **删除用户的最佳实践**：删除用户有两种方式（`DROP USER` 和 `DELETE FROM mysql.user`），为什么文档更推荐使用 `DROP USER`？   

   ​		因为Delete 命令知识删除了mysql.user中的角色，但是mysql.db中会残留刚刚删除角色。使用drop会更干净。

   ​		使用SET

   ​		

   ```mysql
   set password = 'new_password';
   ```

   

5. **密码过期策略**：如果你想设置某个账号（如 `kangshifu`）的密码每 90 天强制过期一次，该如何编写 SQL？   

   ​	

   ```mysql
   Alter user 'user_name'@'user_ip' password EXPIRE INTERVAL 90 DAY;
   ```

   

### 第二部分：权限管理

1. **权限授予原则**：在实际开发中，为什么要遵循“最小权限原则”？请举一个违反该原则的例子。   

   ​	因为最小权限原则最不容易出事故。比如张三的工作只需要查询数据，但是多给了一个修改数据的权限，这就会导致某天张天因为键入错误而导致数据被修改。

   ```mysql
   grant select , insert on atguigudb.* to 'li4'@'localhost';
   ```
