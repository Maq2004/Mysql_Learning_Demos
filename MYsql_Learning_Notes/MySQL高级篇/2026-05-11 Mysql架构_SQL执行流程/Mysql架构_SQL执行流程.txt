### 第一部分：宏观视角 —— MySQL 逻辑架构分层

MySQL 的架构是典型的分层设计，主要分为以下五层。

- **1. 客户端接入层 (Connectors)**
  - **职责：** 提供与不同编程语言交互的 API。
  - **常见组件：** JDBC (Java), Python db-api, PHP PDO 等原生驱动。
- **2. 连接层 (Connection Pool / Management)**
  - **职责：** 负责处理客户端的连接请求、身份验证和安全性校验。
  - **核心机制：** 维护连接池（Connection Pool），管理长连接与短连接，复用线程以降低建立连接的开销。
- **3. 服务层 (Service Layer) —— 【核心计算层】**
  - **职责：** 涵盖 MySQL 的绝大多数核心服务功能，以及所有的跨存储引擎逻辑。
  - **核心组件：**
    - **SQL Interface：** 接收 DML、DDL 等 SQL 指令并返回结果。
    - **Parser (分析器)：** 负责词法与语法分析。
    - **Optimizer (优化器)：** 生成执行计划，决定索引的选择。
    - **Caches & Buffers (缓存)：** （注：MySQL 8.0 版本已彻底移除 Query Cache 查询缓存功能，笔记中需特别标注此版本差异）。
- **4. 引擎层 (Storage Engine Layer) —— 【核心执行层】**
  - **职责：** 负责底层数据的存储和提取。MySQL 采用**插件式存储引擎架构**，服务层通过统一的 API 与引擎层交互。
  - **重点对比：**
    - `InnoDB`：MySQL 5.5 以后的默认引擎。支持事务（ACID）、行级锁、MVCC、外键。
    - `MyISAM`：不支持事务，采用表级锁，查询速度快但并发写入性能弱。
- **5. 存储层 (File System / Storage)**
  - **职责：** 将数据落地到物理磁盘，与操作系统文件系统交互。
  - **核心文件：** 数据文件（.ibd, .MYD）、Redo log（重做日志）、Undo log（回滚日志）、Binlog（归档日志）等。

### 第二部分：微观视角 —— SQL 的完整执行流程

在这部分，我将一条完整的 `SELECT` 查询语句代入架构中，梳理它的生命周期，并明确权限校验的发生节点。

**1. 建立连接 (Connection & Authentication)**

- 客户端通过 TCP 握手与服务端建立连接。
- 连接器验证账号密码。验证通过后，从系统权限表中查出该账号拥有的所有权限，**并将其缓存在当前连接的内存中**（后续所有的静态权限逻辑都基于此缓存）。

**2. 查询缓存 (Query Cache)** *(仅限 MySQL 8.0 之前)*

- 以 SQL 语句为 Key，直接查找缓存。如果命中，直接返回结果。未命中则继续向下执行。

**3. 分析器 (Parser)**

- **词法分析：** 将 SQL 字符串拆解为关键字、表名、列名等 Token。
- **语法分析：** 检查 SQL 是否符合语法规则（如括号是否闭合、关键字是否拼错），最终生成一棵**抽象语法树（AST）**。 

**4. 预处理器 (Preprocessor) —— 【静态权限校验节点】**

- **语义分析：** 检查 AST 中的表名、列名在数据库中是否真实存在。
- **权限校验：** 比对第一步缓存在内存中的权限清单，检查当前用户是否有对目标表和目标字段的操作权限（`precheck`）。无权限则在此处拦截，避免浪费优化器算力。

**5. 优化器 (Optimizer)**

- 基于开销模型（Cost-based Optimizer, CBO）对语法树进行优化。
- 决定使用哪个索引、多表 Join 的连接顺序、是否重写查询等，最终生成一个最优的**执行计划 (Execution Plan)**。

**6. 执行器 (Executor) —— 【动态权限校验节点】**

- 执行器根据优化器生成的执行计划，调用底层存储引擎提供的 API 来获取或修改数据。
- **权限校验：** 针对触发器（Trigger）、视图（View）或存储过程等**只有在运行阶段才能明确涉及哪些表**的操作，执行器会在此时进行运行时的动态权限校验。

**7. 存储引擎层执行 (Storage Engine API)**

- 存储引擎接收执行器的指令，在内存（Buffer Pool）或磁盘中查找对应的数据页。
- 逐行返回符合条件的数据给执行器，执行器进行最后的过滤后组装结果集，通过连接层返回给客户端。
